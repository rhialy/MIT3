-- Die Spalten können in der Abfrage "umbenannt" werden: 
SELECT artist AS Künstler, title AS Lied FROM songs;
-- LIKE kann für eine ungenaue Suche auf Text genutzt werden.

SELECT mit Funktionen in der Abfrage

-- Finden Sie heraus, aus welchen Dateiformaten die Musiksammlung besteht 
-- count() kann Zeilen zählen, In Verbindung mit GROUP BY sehr praktisch! 
SELECT extension, count() FROM songs GROUP BY extension;
-- Welcher Künstler ist am stärksten vertreten? 
-- sum(<Feldname>) kann numerische Felder schon bei der Abfrage aufsummieren. 
SELECT 
  artist,
  count() as tracks, 
  sum(song_length) as length
FROM
  songs
GROUP BY
  artist
ORDER BY
  length desc;
-- Kommandos dürfen auch über mehrere Zeilen gehen! Das kann bei komplexen 
-- Kommandos die Übersicht erhöhen.

Selects über mehrere Tabellen

Die Folgende Abfrage liefert großen Quatsch, wir führen sie dennoch aus:

-- Sie können Daten aus mehreren Tabellen gleichzeitig abfragen: 
SELECT * FROM albums, songs;
-- 19.485.437 Rows returned (took 663ms) <-- !!!!! 19. Millionen. Zeilen. 

Wenn man mehrere Tabellen kombiniert, liefert SQL das "kartesische Produkt" zurück. Das sind alle Zeilen der einen Tabelle kombiniert mit allen Zeilen der anderen Tabelle. Man muss Filtern, um sinnvolle Daten zu erhalten:

-- aus dem (unsinnigen) kartesischen Produkt wählen wir nur die Zeilen, in der Album ID der beiden Tabellen überein stimmt 
SELECT * FROM albums, songs WHERE albums.ID = songs.album_id;
-- 14.063 Rows returned (took 70ms) 
-- auch in kombinierten Abfragen kann man sich auf bestimmte Spalten beschränken. 
-- Wenn der Spaltenname mehrdeutig ist, muss man den Tabellennamen voranstellen (siehe "artist") 
SELECT
  album,
  songs.artist, -- artist kommt doppelt vor, wir möchten den aus der SONGS-Tabelle. 
  title
FROM
  albums,
  songs
WHERE
  albums.ID = songs.album_id;

Wie im einfachen SELECT kann man auch hier noch mehr Bedingungen anfügen:

SELECT
  album, songs.artist, title
FROM
  albums, songs
WHERE albums.ID = songs.album_id
  AND album LIKE "%hits%"; -- NEU: wir filtern zusätzlich nach Albumname 

Aufgabe: Verändern Sie die Abfrage so, dass Alben mit "live" im album-Namen gefunden werden.

-- Selects können auch über noch mehr Tabellen hinweg laufen: 
SELECT *
FROM genres_songs, genres, songs
WHERE genres_songs.genre_id = genres.ID
  AND genres_songs.song_id = songs.ID;

Aus dem letzten Beispiel können wir das Genre eines Lieds ablesen, obwohl es zwei Tabellen entfernt gespeichert ist. genres_songs ist ein sogenannter "Join-Table".

-- und nochmal mit viel mehr Bedingungen 
SELECT *
FROM genres_songs, genres, songs
WHERE genres_songs.genre_id = genres.ID
  AND genres_songs.song_id = songs.ID
  AND genres.genre_name = 'Soundtrack'
  AND songs.title LIKE '%Apollo%';

Artists Beispiel 2 mit Query-Bestandteilen aus Variablen

Im Alltag kommen fast immer Benutzer-Eingaben in die Datenbank-Abfragen. Es ist von großer Wichtigkeit, hier korrekt vor zu gehen, da man sich andernfalls sehr leicht die vielleicht gängigste Sicherheitslücke einfängt, die SQL-Injection.

Bitte bauen Sie sich SQL-Queries nicht mit String-Verkettung (Schlecht: "SELECT * FROM songs WHERE " + boeseBenutzerEingabe + ";") zusammen. Es gibt hierfür sicherere Wege, die ich Ihnen in diesem Beispiel zeigen möchte.

Starten Sie node artists2.js

Gegenüber dem vorherigen Beispiel haben sich nur die Zeilen 5-8 geändert. Besonderes Augenmerk sollten Sie auf Zeile 8 legen. Dort ist im Direktvergleich etwas passiert:

// vorher 
db.all('SELECT * FROM artists', function (err, rows) { ... }
 
// nachher 
db.all('SELECT * FROM artists WHERE artist = ?', suchwort, function (err, rows) { ... }

db.all nimmt Ihnen eine wichtige Aufgabe ab: Wenn Datenbank-Abfragen mit Variablen gefüllt werden müssen (suchwort in unserem Beispiel), kümmert es sich um Anführungszeichen und ein sicheres Einfügen in die Abfrage.

Aufgabe: Verändern Sie die Variable suchwort in Zeile 5 so, dass uns der Eintrag für "They Might Be Giants" ausgegeben wird.

Song Beispiel mit each

Wir haben in den vorigen Beispielen mit all() gearbeitet. Es liefert Ihnen ein Array aller Einträge zurück. Das kann bei sehr umfangreichen Abfragen von Nachteil sein. Eine Alternative ist each(). Im Gegensatz zu bisherigen Beispielen nimmt each() nicht einen sondern zwei callbacks als Parameter. Der erste Callback wird einmal pro Ergebnis-Zeile aufgerufen. Abschließend wird ein zweiter Callback aufgerufen, wenn alles fertig ist.

Für eine bessere Lesbarkeit haben wir in diesem Beispiel auf Fehlerbehandlung verzichtet.

Aufgabe: verändern Sie die Ausgabe in der Konsole so, dass sie etwas menschenlesbarer wird. Das könnte z.B. so aussehen: - Interpret / Titel (ID: 1234).

Der Praktikums-Marathon 2015 neigt sich dem Ende. Wir haben Ihnen die Grundlagen vieler Technologien angeboten, unter Anderem haben wir uns mit HTML, CSS, JavaScript, Templates und SQL beschäfigt. In dieser letzten Aufgabe wird alles zusammen fließen.
